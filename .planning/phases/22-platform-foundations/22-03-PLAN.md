<objective>
Replace all 31 production uses of process.env.HOME with resolveRequiredHomeDir() (or expandHomePrefix() for tilde expansion sites), converting module-level constants to lazy functions where required to preserve call-time evaluation semantics.

Purpose: process.env.HOME is read at the wrong time (module import), is not portable (no OPENCLAW_HOME override, no Windows USERPROFILE fallback), and one site has a hardcoded developer username path ("/Users/user") that would break on any other machine. resolveRequiredHomeDir() from src/infra/home-dir.ts handles all these cases and always returns a string (falls back to process.cwd() if no home dir is set), making it safe for all database path construction.

Output: 19 source files updated, 31 process.env.HOME references removed, test suite still green.
</objective>

```typescript
// src/infra/home-dir.ts — full public API
export function resolveEffectiveHomeDir(
  env?: NodeJS.ProcessEnv, // defaults to process.env
  homedir?: () => string, // defaults to os.homedir
): string | undefined; // returns undefined if no home dir found

export function resolveRequiredHomeDir(
  env?: NodeJS.ProcessEnv, // defaults to process.env
  homedir?: () => string, // defaults to os.homedir
): string; // ALWAYS returns a string — falls back to process.cwd()

export function expandHomePrefix(
  input: string,
  env?: NodeJS.ProcessEnv,
  homedir?: () => string,
): string; // Replaces leading ~ with home dir
```
