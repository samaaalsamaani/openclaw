The schema must create exactly these tables (matching RESEARCH.md Pattern 2):

**files table** — one row per indexed session file:

- `path TEXT PRIMARY KEY` — relative session key path (e.g. "agent:main:telegram:direct:123.jsonl")
- `agent_id TEXT NOT NULL`
- `channel TEXT NOT NULL` — "telegram", "discord", "slack", etc.
- `hash TEXT NOT NULL` — file content hash for incremental skip
- `mtime INTEGER NOT NULL` — file mtime in milliseconds
- `size INTEGER NOT NULL`
- `indexed_at INTEGER NOT NULL`

**chunks table** — one row per text chunk (plan supports 1 chunk per file initially; can extend later):

- `id TEXT PRIMARY KEY` — `<path>:<chunkIndex>` e.g. "agent:main:telegram:...:0"
- `path TEXT NOT NULL` — FK to files.path
- `agent_id TEXT NOT NULL`
- `channel TEXT NOT NULL`
- `start_line INTEGER NOT NULL`
- `end_line INTEGER NOT NULL`
- `hash TEXT NOT NULL`
- `text TEXT NOT NULL` — the indexed text content
- `updated_at INTEGER NOT NULL`

Note: NO embedding column for now — the initial implementation uses FTS5 full-text search only (no vector embedding dependency). This keeps the indexer dependency-free while delivering MEM-01 and MEM-05. Plan 02 uses FTS5 search.

**fts virtual table** (FTS5 external content):

```sql
CREATE VIRTUAL TABLE IF NOT EXISTS chunks_fts USING fts5(
  text,
  content='chunks',
  content_rowid='rowid',
  tokenize='trigram'
)
```

**Indexes**:

- `idx_chunks_channel ON chunks(channel)`
- `idx_files_channel ON files(channel)`
- `idx_files_agent_id ON files(agent_id)`

After creating each table, also ensure the FTS table is populated via a trigger or inline:

```sql
CREATE TRIGGER IF NOT EXISTS chunks_ai AFTER INSERT ON chunks BEGIN
  INSERT INTO chunks_fts(rowid, text) VALUES (new.rowid, new.text);
END;
CREATE TRIGGER IF NOT EXISTS chunks_ad AFTER DELETE ON chunks BEGIN
  INSERT INTO chunks_fts(chunks_fts, rowid, text) VALUES ('delete', old.rowid, old.text);
END;
CREATE TRIGGER IF NOT EXISTS chunks_au AFTER UPDATE ON chunks BEGIN
  INSERT INTO chunks_fts(chunks_fts, rowid, text) VALUES ('delete', old.rowid, old.text);
  INSERT INTO chunks_fts(rowid, text) VALUES (new.rowid, new.text);
END;
```

Use `db.exec()` for each statement (same pattern as src/memory/memory-schema.ts). No external dependencies — only `node:sqlite` types.

**Exports required:**

```typescript
export type CrossChannelSearchResult = {
  path: string; // session key path (relative)
  channel: string; // "telegram", "discord", etc.
  mtimeMs: number; // file mtime for relative time display
  snippet: string; // matched text excerpt
  score: number; // FTS5 BM25 rank (lower = better; negate for descending)
};

export class CrossChannelIndexer {
  static getInstance(agentId: string): CrossChannelIndexer;
  start(): void; // starts periodic 5-min indexer interval (idempotent)
  stop(): void; // clears interval, closes DB (for tests)
  sync(): Promise<void>; // run one indexer pass immediately (also called by interval)
  search(params: {
    query: string;
    excludeChannel?: string; // current channel — exclude from results
    maxResults?: number; // default 5
    minScore?: number; // FTS5 BM25 rank filter (default: no filter)
    charBudget?: number; // max total chars of snippets (default 4000)
  }): CrossChannelSearchResult[];
}
```

**DB path:** `path.join(resolveRequiredHomeDir(), ".openclaw", "agents", agentId, "cross-channel-memory.sqlite")`
(NOT inside workspace dir — must be at the agent level)

**Singleton pattern:** Use a module-level `Map<string, CrossChannelIndexer>` (keyed by agentId), same as the `INDEX_CACHE` in `src/memory/manager.ts`:

```typescript
const INDEXER_CACHE = new Map<string, CrossChannelIndexer>();
// getInstance() checks INDEXER_CACHE first, creates if missing
```

**`sync()` implementation:**

1. Call `listSessionFilesForAgent(agentId)` to get all abs paths
2. For each abs path, derive the session key: `path.basename(absPath, ".jsonl")`
3. Skip files where `isCronRunSessionKey(key)`, `isCronSessionKey(key)`, or `isSubagentSessionKey(key)` is true
4. Extract channel using `parseAgentSessionKey(key)?.rest.split(":")[0]?.trim().toLowerCase() ?? null`
5. Skip files where channel is null or `"unknown"`
6. Call `buildSessionEntry(absPath)` — returns null if file is empty/unreadable (skip those too)
7. Hash-compare: `db.prepare("SELECT hash FROM files WHERE path = ?").get(entry.path)` — skip if hash unchanged
8. Filter slash commands from content: split on `\n`, remove lines matching `/^User:\s*\//`, rejoin
9. If content is empty after filtering, skip (don't index empty sessions)
10. Write to `files` table (upsert via `INSERT OR REPLACE`)
11. Delete old chunks for this path: `DELETE FROM chunks WHERE path = ?`
12. Insert single chunk: id = `${entry.path}:0`, start_line = 0, end_line = entry.lineMap.length - 1
13. After processing all active paths, purge stale entries:
    - `SELECT path FROM files WHERE agent_id = ?` → compare to activePaths set
    - DELETE from `files` and `chunks` where path no longer active
14. Emit observability event: action = `"cross_channel_index_sync"`, metadata = `{ filesIndexed, filesSkipped, channelsSeen: [...unique channels], agentId, durationMs }`

**`search()` implementation:**
Use FTS5 full-text search via the `chunks_fts` virtual table:

```sql
SELECT c.path, c.channel, f.mtime, c.text, bm25(chunks_fts) AS rank
FROM chunks_fts
JOIN chunks c ON chunks_fts.rowid = c.rowid
JOIN files f ON c.path = f.path
WHERE chunks_fts MATCH ?
  AND c.agent_id = ?
  AND c.channel != ?   -- exclude current channel (pass "" if no exclusion needed)
ORDER BY rank           -- BM25: lower is better
LIMIT ?
```

Apply charBudget truncation (same logic as `clampResultsByInjectedChars` in memory-tool.ts):
iterate results, accumulate snippet char count, stop when budget exceeded.

**`start()` implementation:**
Guard with a module-level `let started = false` per instance (not global — keyed per agentId instance):

```typescript
start(): void {
  if (this._started) return;
  this._started = true;
  this._interval = setInterval(() => { void this.sync(); }, 5 * 60 * 1000);
  // Run immediately on start
  void this.sync();
}
```
